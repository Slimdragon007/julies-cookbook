- **TASK-009 — Service worker no longer hides newly-added recipes from the gallery.** Done 2026-04-28. Surfaced when Julie added a honey-mustard recipe via URL, the API returned `success: true`, but the recipe never appeared in her gallery. DB audit confirmed the recipe was saved correctly under Julie's `user_id` with image + 9 ingredients — persist was innocent. Root cause: `public/sw.js` cached all HTML pages with stale-while-revalidate (returns cached first, fetches fresh in background) and **zero `caches.delete()` calls existed anywhere in the React app to bust the cached gallery after a successful save**. Fix: bumped `CACHE_NAME` from `cookbook-v2` → `cookbook-v3` (forces all clients to evict the stale cache on activate); switched HTML/page navigations from SWR to network pass-through; static assets and Cloudinary images keep their cache-first strategies (perf preserved). Added `router.refresh()` in `AddRecipeForm` after both URL- and text-submit success to evict the Next.js router cache as well. Lint clean, tsc clean. See `@progress.md`.
